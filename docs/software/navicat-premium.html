# Navicat Premium

Navicat Premium软件是一个数据库远程连接管理软件，其中包含了MySQL、PostgreSQL、Oracle、SQLite、SQL Server、MariaDB、MongoDB等主流数据库的远程连接功能。除此之外，Premium还集成了阿里云、腾讯云以及华为云数据库连接，是数据库管理软件中的翘楚，具有使用便利，界面友好，运行流畅等关键性优点。

# 1.安装

  • 双击软件安装程序文件打开，依次点击下一步按钮进行软件安装，安装成功后在桌面打开已经安装好的Navicat Premium 15软件，第一次打开软件的时候回提示试用提醒，如下图所示。

  • 点击注册按钮，此刻会弹出如下图所示的需要添加许可证秘钥激活的窗口，进入软件激活步骤。

# 2.激活

  • 右键注册软件Navicat Keygen Patch v5.6.0 DFoX.exe，以管理员身份运行，注册机软件打开后点击软件界面右上角的Patch按钮，之后会弹出如下图所示的窗口，提示Cracked，表示第一步已经成功，点击确定按钮之后会关闭此信息框，可以进行第二步。如果这个过程弹出了其他信息框，说明出现了Patch错误，需要到标题3的问题栏中找解决方案。

  • 注册机界面中的2和3步选择默认配置，不可调整，如下图所示。直接跳到第4步中的Keygen/Offine Activation，点击Generate按钮会生成Keygen并自动填充到软件中，修改使用者的名字和组织，这两项可以任意修改，点击软件注册页面的激活按钮，这时会提示弹窗，选择手动激活。

  • 在手动激活窗口中，将软件生成的请求码复制到注册软件中的Request Code输入框中

  • 点击Activation Code下的Generate按钮，可以是生成激活码，激活码会自动复制到软件的手动激活窗口下的激活码输入框中。

  • 点击手动激活窗口中的激活按钮，此刻会弹出Navicat现已激活的对话框，点击确认即可完成激活。

# 3.安装问题

# 1.出现“Rsa public key not find”

    1. 检查是否未关闭Navicat Premium。
    2. 或到安装目录下将libcc.dll和navicat.exe删除，并将libcc.dll.BAK或navicat.exe.BAK去掉.BAK后缀名。

    然后再次操作。 引起这个问题是在点击patch按钮时没有生成破解版的RegPrivateKey.pem 即路径C:\Users{用户名}\AppData\Local\Temp下不存在RegPrivateKey.pem
